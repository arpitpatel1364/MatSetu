Security Rules

All 10 absolute security constraints enforced in this MatSetu instance

Absolute Constraints
SEC-1vote_ledger is APPEND-ONLY. UPDATE and DELETE are REVOKED at PostgreSQL DB role level. Trigger enforced as belt-and-suspenders.
SEC-2voters.has_voted once TRUE is IMMUTABLE. No code path may reset it. Database trigger prevents reversal at row level.
SEC-3All vote operations are fully atomic: Redis MULTI/EXEC pipeline + PostgreSQL transaction execute together or not at all.
SEC-4Row Level Security is always ON. Every query respects scope_id and scope_type from JWT claims. Cross-scope reads blocked.
SEC-5OTPs expire in 5 minutes. Max 3 attempts. OTPs stored exclusively as SHA-256(otp) — plaintext never persisted.
SEC-6ZK receipts never reveal candidate — only inclusion proof (Helios-style commitment). Receipt verifies presence in ledger only.
SEC-7JWT tokens are role-scoped. Cross-scope use triggers immediate 403 REJECT. Scopes: all_india, one_state, one_div, one_dist, one_const, one_booth.
SEC-8All manual admin overrides trigger ANOMALY_OVERRIDE entry in immutable audit_log with actor ID, TOTP confirmation and reason.
SEC-9Election start/stop and UNCONTESTED declaration require TOTP re-confirmation at time of action, even for authenticated sessions.
SEC-10Master Admin (T1) has read-only on vote_ledger via RLS pass-through. No INSERT, UPDATE, or DELETE ever permitted for T1.

Election Control

T1 exclusive — SEC-9 TOTP required for start and stop actions

Start Election

Opens all PENDING constituencies. Activates eligible booths. Marks election_status=OPEN. UNCONTESTED seats excluded.

Stop Election

Closes all OPEN constituencies. Deactivates all active booths. Records ELECTION_STOPPED in audit log. Requires TOTP.
